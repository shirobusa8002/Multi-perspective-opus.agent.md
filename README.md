## 概要

**Multi-perspective-opus.agent.md** は、VS Code / GitHub Copilot のエージェントモード（`.github/` 配下などに配置するカスタムエージェント指示ファイル）向けの **分析フレームワーク** です。

Claude Opus 4.6 の拡張思考（最大128Kトークン）を戦略的に活用し、あらゆるテーマ・課題を **6つの異なる視点 + メタ認知層** から客観的・論理的に深掘り分析することで、**高品質な意思決定** を支援します。

---

## 何ができるのか？

### 🎯 6つの視点からの多角的分析

1つの課題を、以下の6つの専門的立場から同時に分析します：

| 視点 | 役割 | 主な評価軸 |
|------|------|----------|
| 👤 **ユーザー代表** | エンドユーザーの立場を代弁 | UX/UI、パフォーマンス、価格妥当性、信頼性 |
| 💻 **開発者代表** | 技術実装の立場を代弁 | コード品質、アーキテクチャ、技術的負債、DX |
| 🏢 **運用企業代表** | ビジネス・運用の立場を代弁 | 収益性、ROI、コンプライアンス、競争優位性 |
| 🔒 **セキュリティ代表** | 情報セキュリティの立場を代弁 | 脆弱性対策、アクセス制御、監査ログ、STRIDE分析 |
| 📊 **データ代表** | データ品質・活用の立場を代弁 | データガバナンス、品質メトリクス、ML/AI公平性 |
| 🌍 **倫理・社会影響代表** | 倫理的・社会的影響の立場を代弁 | 公平性、透明性、環境負荷、世代間影響 |

### 🧠 Opus 4.6 固有の高度な分析機能

| 機能 | 説明 |
|------|------|
| **トゥールミンモデル論証** | 主張・根拠・論拠・裏付け・限定条件・反証条件を明示した構造化論証 |
| **確信度キャリブレーション** | 全ての結論に確信度（%）を付与し、不確実性を定量化 |
| **反事実分析** | 「もし〜でなければどうなるか？」による前提の感度検証 |
| **3シナリオモデリング** | 楽観・基準・悲観の3パターンで影響をシミュレーション |
| **メタ認知検証** | 認知バイアス監査、悪魔の代弁者テスト、推論チェーン検証 |
| **Premortem分析** | 「失敗した未来」を想定した事前検死で撤退基準・早期警戒指標を設定 |
| **感度分析** | 視点の重み付けを変動させた場合の結論安定性を検証 |

### 📋 4段階の分析フロー

```
🧠 プレラウンド ─→ 📋 第1ラウンド ─→ 🔄 第2ラウンド ─→ ⚖️ 第3ラウンド ─→ 🔬 検証ラウンド ─→ 🎯 最終レポート
  問題の構造化分解     深層初期分析     深層相互検証      統合と最適解導出    メタ認知的検証      結論と推奨
```

| ラウンド | 内容 | 拡張思考の使用量 |
|---------|------|----------------|
| **プレラウンド** | 問題の原子的分解、因果グラフ構築、隠れた前提の明示化 | — |
| **第1ラウンド** | 各視点から独立して深層分析（トゥールミンモデル準拠） | ~25K |
| **第2ラウンド** | 視点間の相互検証、反事実分析、3シナリオ分析 | ~40K |
| **第3ラウンド** | 6視点を統合した最適解の導出、感度分析 | ~25K |
| **検証ラウンド** | 認知バイアス監査、悪魔の代弁者テスト、Premortem | ~30K |

## どう使うのか？

### 📥 セットアップ

1. **リポジトリをクローン**

   ```bash
   git clone https://github.com/shirobusa8002/Multi-perspective-opus.agent.md.git
   ```

2. **エージェントファイルを配置**

   `Multi-perspective-opus.agent.md` ファイルを、分析したいプロジェクトの `.github/` ディレクトリなど、VS Code / GitHub Copilot Agent が読み込める場所にコピーします。

   ```bash
   # 例：自分のプロジェクトにコピー
   cp Multi-perspective-opus.agent.md <your-project>/.github/
   ```

3. **VS Code で開く**

   VS Code の GitHub Copilot（Agent モード対応版）で、エージェントファイルを読み込ませた状態でチャットを開始します。

### 🚀 分析を始める

エージェントモードのチャットで、分析したいテーマを伝えるだけで自動的に分析が開始されます。

#### 基本的な使い方

```
「AIチャットボット機能を自社サービスに導入すべきか検討したい」
```

```
「モノリスからマイクロサービスへの移行を検討している」
```

```
「新しい認証基盤の選定について多角的に分析してほしい」
```

#### 分析モードの選択

時間制約や意思決定の重大度に応じて、4つのモードから選択できます：

| モード | ラウンド数 | 所要時間目安 | 適したケース |
|--------|-----------|-------------|-------------|
| ⚡ **簡易** | 1ラウンド | 5-10分 | 方向性の確認、クイックチェック |
| 📋 **標準** | 3ラウンド | 15-30分 | 重要な意思決定 |
| 🔬 **深層**（デフォルト） | 3+検証 | 30-60分 | 不可逆的な決定 |
| 🔭 **徹底** | 3+検証+感度 | 60分以上 | 極めて重大な決定 |

#### 視点の重み付けカスタマイズ

デフォルトでは6視点均等ですが、シナリオに応じて重み付けを変更できます：

### 📄 分析レポートの出力

分析完了後、構造化されたレポートが自動生成されます：

- **保存先**: `.github/analyses/analysis-YYYY-MM-DD-[テーマ]-opus.md`
- **含まれる内容**:
  - エグゼクティブサマリー（最終確信度付き）
  - 各視点の評価サマリー
  - トレードオフと判断根拠
  - 3シナリオ分析結果
  - 確信度付きアクションプラン（P1〜P5の優先度）
  - リスク対策と早期警戒指標
  - 撤退基準（Exit Criteria）
  - KPI・成功指標
  - メタ認知検証結果

### 🔄 継続分析・更新

同一テーマの分析ドキュメントが存在する場合、既存の分析を引き継いで差分分析が可能です：

```
「前回のAIチャットボット分析の続きをしたい」
```

## 使用例

### 例1: 新機能の導入検討

**質問**: 「AIチャットボット機能を自社サービスに導入すべきか」

**分析の流れ**:

- 🧠 **プレラウンド**: 導入の因果グラフ構築、ユーザーのAI受容性・データ充足性などの隠れた前提を明示化
- 📋 **第1ラウンド**: 各視点から確信度付きの独立分析（例: コスト30%削減見込み[確信度60%]、プロンプトインジェクションリスク[確信度90%]）
- 🔄 **第2ラウンド**: 反事実分析「もしユーザーがAI対応を拒否したら？」、3シナリオ分析（ROI: 楽観250% / 基準120% / 悲観-30%）
- ⚖️ **第3ラウンド**: FAQ対応からの段階導入を統合提案、感度分析で結論の頑健性を確認
- 🔬 **検証ラウンド**: 楽観バイアス検出→修正、Premortem で最大失敗要因（データ品質不足）を特定
- 🎯 **結論**: 限定的導入から開始（確信度78%）、撤退基準付き

### 例2: アーキテクチャ刷新

**質問**: 「モノリスからマイクロサービスへの移行を検討」

**分析の流れ**:

- 反事実分析で「モノリスのまま最適化」の代替案を定量評価（コスト1/3で70%の効果）
- 新技術バイアスを検出し、モノリス最適化の再評価を実施
- **結論**: ストラングラーパターンによる段階的移行を推奨（確信度72%）

## Sonnet 4.5版との主な違い

| 項目 | Sonnet 4.5版 | Opus 4.6版（本エージェント） |
|------|-------------|---------------------------|
| 視点数 | 5視点 | **6視点 + メタ認知層** |
| 拡張思考 | 最大64Kトークン | **最大128Kトークン** |
| 推論深度 | 3ラウンド | **3ラウンド + 検証ラウンド** |
| 確信度評価 | なし | **ベイズ的確信度キャリブレーション** |
| 反事実分析 | なし | **標準装備** |
| バイアス検出 | 注意事項レベル | **構造化バイアス検出プロトコル** |
| シナリオモデリング | なし | **3シナリオ分析（楽観・基準・悲観）** |
| 論証構造 | 根拠明示 | **トゥールミンモデル準拠** |
